# 2. Types of Databases

Different applications require different ways of storing data.

---

## 1. Relational Database (RDBMS)

- Stores data in tables.
- Tables are related using keys.
- Uses SQL.
- Follows a fixed schema.

### Examples
- MySQL
- PostgreSQL
- Oracle
- SQL Server

### Used in
- Banking
- College management
- ERP systems

---

## 2. NoSQL Database

- Non-relational database.
- Flexible schema.
- Suitable for unstructured or semi-structured data.

Types:
- Key-Value
- Document
- Column Family
- Graph

### Examples
- MongoDB
- Cassandra
- Redis

### Used in
- Social media
- Real-time applications
- Big data

---

## 3. Object-Oriented Database

Stores data as objects instead of tables.

Useful for object-oriented programming applications.

---

## 4. In-Memory Database

Stores data mainly in RAM instead of disks.

Advantages:
- Extremely fast
- Low latency

Applications:
- Real-time analytics
- Caching

---

## 5. Time-Series Database

Stores data associated with timestamps.

Applications:
- Stock prices
- IoT sensors
- Weather monitoring

---

## 6. Spatial Database

Stores geographical and geometric data.

Applications:
- Google Maps
- GPS systems
- GIS applications

---

## 7. Multimedia Database

Stores multimedia data.

Examples:
- Images
- Audio
- Videos
- Graphics

Applications:
- Streaming platforms
- Medical imaging

---

## 8. Columnar Database

Stores data column-wise instead of row-wise.

Advantages:
- Faster analytical queries
- Better compression

Applications:
- Data warehouses
- Business intelligence

---

## 9. XML Database

Stores data in XML format.

Useful when applications exchange XML documents.

---

## 10. NewSQL Database

Combines:
- Relational database features
- SQL support
- High scalability similar to NoSQL

Applications:
- Distributed financial systems

---

## 11. Blockchain Database

Stores data in linked blocks.

Characteristics:
- Decentralized
- Immutable
- Highly secure

Applications:
- Cryptocurrencies
- Supply chain
- Smart contracts

---
